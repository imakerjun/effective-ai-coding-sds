# Contributing

## 한글 환경에서 자주 깨지는 Markdown 패턴

### 강조(`**bold**`) + 한글

CommonMark 강조 마커는 앞뒤 문자가 "단어 경계"인지 본다. 닫는 `**` 직전에 구두점(`)`, `"`, `'`)이 있고 직후에 한글이 바로 붙으면, 일부 파서가 닫는 emphasis로 인식하지 못하고 `**` 를 그대로 출력한다.

```text
❌ 답은 **하네스(harness)**입니다.        → 본문에 ** 가 그대로 보임
✅ 답은 **하네스(harness)** 입니다.       → 공백 1칸으로 안전
✅ 답은 **하네스 (harness)** 입니다.      → 괄호 앞 공백으로도 가능
✅ 답은 _하네스(harness)_ 입니다.         → italic 사용도 안전
```

**규칙**: 닫는 `**` 다음에 한글을 붙여 쓰지 말고, 항상 공백을 한 칸 둔다.

### 자동 검사

```bash
npm run check:md
```

문서를 커밋하기 전에 실행해 위험 패턴이 없는지 확인한다. CI에 추가하려면 동일 명령을 빌드 단계 앞에 박으면 된다.

## Mermaid 다이어그램

- **timeline**: 한글 큰따옴표 (`"`) 사용 금지. 평문으로 작성.
- **quadrantChart**: 데이터 포인트 라벨에 한글 사용 금지. 영문 라벨 + 본문에 한글 매핑 표를 두는 패턴 권장.
- 신규 다이어그램 추가 시 Playwright 콘솔 에러 0건 검증.

## 기타

- 강의 콘텐츠 톤은 구어체 기반.
- 챕터 번호는 파일명 접두사(`01-`, `02-` ...)로 명시. `_meta.ts`만으로 순서를 관리하지 않는다.
- 사례 출처는 본문에 `> 출처: [...]` blockquote로 명시.
