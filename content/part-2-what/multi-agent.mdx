# 2.5 멀티 에이전트 활용

> 역할을 나누는 힘

## 왜 멀티 에이전트인가

단일 에이전트로 모든 걸 시키면:
- 컨텍스트 오염
- 역할 혼재
- 검증 부재

역할을 나누면 각 에이전트가 한 가지에만 집중할 수 있다.

## 3가지 패턴

### 패턴 1: Plan / Implement / Review 분리

- **Plan Agent**: 요구사항 → 작업 계획
- **Implement Agent**: 계획대로 구현
- **Review Agent**: 독립적으로 결과 검증

### 패턴 2: Main + 서브에이전트

- Main은 전체 흐름 관리
- 서브에이전트는 탐색·검증·요약 같은 "컨텍스트 많이 먹는 일" 전담
- → Main의 컨텍스트 윈도우 보호

### 패턴 3: 에센스 정의 → 변형 생성 (디자인팀 패턴)

- 하나의 "에센스(기준)"를 정의
- 여러 에이전트가 같은 에센스를 참조해 일관된 변형 생성

- 스컬피 원형 = CLAUDE.md, 코딩 컨벤션, 테스트 템플릿
