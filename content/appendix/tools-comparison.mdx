## 어떤 걸 골라야 할까

### Claude Code를 추천하는 경우

- **이 강의의 5가지 원칙을 가장 매끄럽게 적용**할 수 있음
- Plan Mode, 서브에이전트, 훅이 **네이티브** 기능
- CLAUDE.md 컨벤션이 가장 잘 자리잡음
- 강의 데모·스크린샷이 Claude Code 기준

### Codex를 추천하는 경우

- 이미 OpenAI 생태계 (ChatGPT Pro/Team) 사용 중
- AGENTS.md 컨벤션을 다른 도구와 공유하고 싶음

### AI Pro를 써야 하는 경우 ⭐

- 삼성SDS 사내 정책상 외부 도구 사용 제한
- 사내 인프라·데이터 접근이 필요한 작업
- **JetBrains/VSCode/Eclipse 셋 중 어디서든 동일 경험**을 원하는 경우 (PDF 가이드에 세 IDE 모두 명시)

**AI Pro만의 장점**:
- IDE에 깊게 통합된 컨텍스트 메뉴 (Code Generation / Refactoring / Comment / Test / Explanation)
- 한 번에 여러 모델 선택 가능 (Qwen3-Coder, GPT-OSS, GLM-4.7 등)
- Skill 자체를 만들어주는 메타스킬 `skill-creator` 빌트인

**AI Pro만의 제약**:
- Rules 합산 10,000자 제한 (Claude Code는 사실상 무제한)
- MCP는 아직 Beta
- 외부 패키지 자유도는 Claude Code/Codex보다 좁을 수 있음

## 도구가 달라도 원칙은 같다

이 강의의 핵심 메시지입니다.

| 원칙 | Claude Code | Codex | **AI Pro** |
|---|---|---|---|
| Context | `CLAUDE.md` | `AGENTS.md` | **Rules** (`.aipro/rules/*.md`) + `AGENTS.md` |
| Plan | Plan Mode | "계획 먼저" 프롬프트 | **Chat ↔ Agent 모드 + Pre-set Prompt** |
| Token | `/compact` + 서브에이전트 | 세션 분할 | **Chat History 압축 + `.aiproignore` + Workspace Indexing** |
| Quality | Hook + Test | Test + 사람 리뷰 | **Editor → Create Test / Code Explanation / Commit Message** |
| Multi-Agent | 서브에이전트 + Task | 세션 분리 | **Skills** (`/skill-name` + 자동 트리거) + skill-creator |

**원칙은 5개, 구현은 도구마다 다름.** 원칙을 알면 어떤 도구로 옮겨도 적응이 빠릅니다.

## 도구 전환 시 체크리스트

도구를 바꿔야 한다면:

- [ ] 컨텍스트 파일 이름과 위치 (CLAUDE.md ↔ AGENTS.md)
- [ ] Plan Mode 진입 방법
- [ ] 서브에이전트/Task 호출 방법
- [ ] 훅 또는 자동화 방법
- [ ] 권한·승인 흐름

이 5가지만 매핑하면 도구 전환 비용이 크게 줄어듭니다.
